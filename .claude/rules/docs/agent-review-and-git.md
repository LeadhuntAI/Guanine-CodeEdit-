## Purpose
Flask Blueprint controller and supporting services for the agent code review system. Provides hunk-level code review where multiple agents modify files and changes are visualized in a combined diff view with color-coded attribution. Supports two agent backends (built-in OpenRouter/ReAct and external OpenCode server), git operations (clone, pull, push, deploy), SSH key management, and real-time chat streaming via SSE.

## Data Flows
- **Repo registration:** User submits repo config via `register_repo()` → stored via `agent_schema` → workspace created on disk.
- **Session lifecycle:** `create_session()` → backend selected via `get_backend_for_repo()` → `start_session()` spawns agent in sandbox → events streamed via SSE (`api_chat_events`) → session status tracked in DB.
- **Review workflow:** Agent completes task → `_create_review_session()` builds `MergeItem` inventory (Original vs Agent versions) via `file_merger` → user reviews hunks → `api_review_decision()` records accept/reject → `_execute_merge_to_repo()` copies accepted files to original repo.
- **Multi-agent diff:** `generate_multi_agent_diff()` reads original + per-agent file versions → computes per-agent change sets → groups overlapping changes into conflict hunks → returns combined hunk list with agent attribution.
- **Git deploy:** `run_deploy()` reads repo settings → executes SSH command on remote host via `_run_ssh()`.

## Invariants Enforced
- Merge operations require session status to be `'review'` before proceeding (`_execute_merge_to_repo`).
- Review sessions are created only from sessions with modified files (`_create_review_session` raises `ValueError` if none).
- OpenCode server ports are allocated from range `[4096, 4200)` with lock-based mutual exclusion (`_allocate_port`).
- SSH key generation uses ED25519 algorithm exclusively.
