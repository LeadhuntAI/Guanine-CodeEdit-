## What This Project Does
Guanine is a Flask-based web IDE that combines multi-source file merging with an AI-powered code review system. It scans multiple source directories, detects file conflicts, presents side-by-side diffs, and lets users interactively resolve merges. Separately, it provides a sandboxed environment where AI coding agents (via MCP or built-in OpenRouter backend) check out files, make modifications, and submit changes for human review with hunk-level, color-coded diff attribution across multiple agents.

## Features

### Multi-Source File Merging
Scans multiple source directories, builds a merged inventory categorized as auto-unique, auto-identical, or conflict, then presents conflicts in a side-by-side diff editor for interactive resolution. Files are copied to a target directory with progress streamed to the browser via SSE.

**How it works:** `FileScanner` computes SHA-256 hashes and metadata per file, `build_inventory()` groups by relative path and categorizes, then `MergeEngine.execute_merge()` copies resolved files through a `queue.Queue` that feeds SSE events to the frontend.

## Architecture Overview
The Flask server (`file_merger.py`) is the entry point, mounting the agent review Blueprint and serving all routes. The merge engine operates independently for file scanning and conflict resolution. The agent system centers on `agent_session_schema` as the single source of truth, with `agent_sandbox_tools` providing the execution environment, `agent_mcp_server` adapting it for external clients, and `agent_review` providing the web review workflow. The `agentic_engine` handles LLM orchestration and dispatches to `agentic_tools` for file operations. All UI is rendered through Jinja2 templates with a VS Code-style IDE shell.
