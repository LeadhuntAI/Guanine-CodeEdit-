## Purpose
Contains all Jinja2 HTML templates and static assets for the Guanine application. Provides two distinct UI surfaces: (1) a **core recovery workflow** (setup, inventory, conflicts, merge) rendered by Flask routes in `file_merger.py`, and (2) a **full VS Code-style IDE shell** (`ide_shell.html`) with integrated file browsing, CodeMirror 6 editor, inline diff review, agent chat panel, and dashboard — rendered by routes in both `file_merger.py` and `agent_review.py`.

## Data Flows

- **Template context → HTML**: Flask routes pass Python dicts (file trees, stats, session data, hunks) as Jinja2 context; templates render them into HTML.
- **User actions → API calls**: Templates embed JavaScript that calls REST/SSE endpoints (`/api/browse-tree`, `/api/save-merge`, `/agent/api/*`, `/merge-events`, `/scan-events`) to read/write data without full page reloads.
- **IDE shell state → localStorage**: `ide_shell.html` persists open tabs, sidebar width, and active project to `localStorage` under key `guanine_ide_session`.
- **SSE streams → UI**: Scan progress, merge progress, and agent chat messages flow via Server-Sent Events into the IDE's bottom panel and chat panel.

## Invariants Enforced

- All page templates extend `base.html` (except `ide_shell.html` which is standalone).
- `ide_shell.html` loads CodeMirror 6 as an async ESM module from `static/codemirror.js`; the editor waits for a `cm-ready` custom event before initializing.
- Inline diff review auto-saves when all hunks are decided (accept/reject/edit).
- Unsaved editor changes trigger `beforeunload` warnings in both the merge editor and IDE shell.

## Shared Types

Templates expect these context variable shapes (defined in Python backends, consumed here):

- **`stats` dict**: `total`, `auto_unique`, `auto_identical`, `conflicts`, `resolved`, `auto_merge`
- **`items` list**: Each item has `relative_path`, `category` (`auto_unique`/`auto_identical`/`conflict`), `resolved`, `source_names`, `size`, `modified`, `lines`, `num_versions`, `selected_source`
- **`hunks` list**: Each hunk has `type` (`equal`/`conflict`), `id`, `start_a`, `start_b`, `lines_a`, `lines_b`, `tag` (`replace`/`delete`/`insert`)
- **`sessions` list**: Each session has `session_id`, `task`/`task_description`, `status`, `files`, `modified_count`, `backend`
- **`agent_session_id`**: Optional string; when present, IDE toolbar shows "Merge Resolved" / "Accept All & Merge" buttons instead of "Auto-Merge" / "Merge All"

## Configuration

- **CDN dependencies**: Bootstrap 5.3.3 CSS/JS, highlight.js 11.9.0 (loaded in `ide_shell.html` head)
- **CodeMirror 6**: Loaded from local `static/codemirror.js` as ESM module; rebuild command: `npx esbuild _cm_bundle_entry.js --bundle --format=esm --minify --outfile=static/codemirror.js`
- **localStorage key**: `guanine_ide_session` for tab/session persistence; `guanine_active_repo` for active project

## Impact Notes

- **Changing `base.html`** affects all 15+ page templates that extend it (everything except `ide_shell.html`).
- **Changing `ide_shell.html`** is high-risk: it is a 5300-line standalone page containing the entire IDE frontend. CSS classes, JS function names, and DOM element IDs are referenced throughout.
- **Changing `_chat_panel.html`** or `_dashboard.html`** breaks the IDE shell's chat and dashboard sidebar views.
- **Removing or renaming template context variables** (e.g., `stats`, `hunks`, `sessions`) will cause Jinja2 `UndefinedError` at runtime — must be coordinated with `file_merger.py` and `agent_review.py` route handlers.
- **Changing API endpoint URLs** in template JavaScript (e.g., `/api/save-merge`, `/agent/api/sessions-browse`) must be coordinated with the corresponding Flask route definitions.
