## Purpose

This area contains the complete documentation suite, templates, and example definitions for the agentic workflow engine. It follows a **documentation-as-configuration** pattern: markdown files with structured YAML frontmatter drive runtime behavior (skill discovery, knowledge injection) while serving as human-readable reference. Each doc is self-contained ŌĆö agents and developers should not need to read engine source code to create workflows, skills, or tools.

## Data Flows

- **Knowledge injection pipeline:** YAML-frontmatter markdown files ŌåÆ `extract_frontmatter()` ŌåÆ `build_rules_index()` / `build_skills_index()` ŌåÆ XML-tagged blocks (`<AVAILABLE_RULES>`, `<AVAILABLE_SKILLS>`) injected into agent system prompts
- **Skill discovery:** `SKILL.md` files in `agentic/skills/*/` and JSON files in `agentic/skill_definitions/` ŌåÆ `discover_skills()` ŌåÆ metadata list ŌåÆ `<AVAILABLE_SKILLS>` index
- **Workflow execution:** JSON workflow definitions ŌåÆ `run_workflow()` ŌåÆ `LightweightWorkflowSession` ŌåÆ sequential layer processing ŌåÆ accumulated context and outputs
- **Prompt processing:** Layer prompts ŌåÆ Jinja2 rendering (`{{ variable }}`) ŌåÆ placeholder replacement (`{PLO}`, `{OutputLayer-N}`) ŌåÆ LLM call or template output

## Invariants Enforced

- **Self-contained documentation:** All docs explicitly instruct readers not to open engine source code; the documented JSON structures and formats are the authoritative reference.
- **YAML frontmatter required fields:** Every rule/doc/skill markdown file must have `name` and `description` in its frontmatter. Fallback parsing extracts first `#` heading as name and first paragraph as description if frontmatter is missing.
- **Tool convention:** Every tool module must expose an `execute()` function returning a JSON string, accepting `_base_dir` and `**kwargs`.
- **Progressive disclosure:** `rules` are always fully injected; `knowledge_set` and `skills` show summaries only, with full content loaded on demand.

## Impact Notes

- **Changing doc examples or templates** could break user-created workflows/skills that follow the documented patterns. The docs are the authoritative reference ŌĆö any format change must be reflected in both docs and engine code.
- **Modifying YAML frontmatter fields** (e.g., adding a new required field) would require updates in `agentic/engine/knowledge.py`'s `extract_frontmatter()` and all existing SKILL.md / rule files.
- **Changing layer type definitions** in `json-definitions.md` would need corresponding changes in `agentic/engine/runner.py`'s `process_layer()` dispatch.
- **The self-contained documentation principle** means agents rely solely on these docs ŌĆö if docs become stale relative to engine code, agent behavior will be incorrect.
