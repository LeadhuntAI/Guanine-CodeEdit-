## Purpose
Flask-based web IDE for multi-source file merging. Provides file scanning and inventory building across multiple source directories, conflict detection and categorization, side-by-side diff visualization, interactive merge editing, and SQLite-backed session persistence. This is the main entry point for the application.

## Data Flows
- **Source directories** → `FileScanner.scan_source()` → per-file `FileVersion` dataclasses with SHA-256 hashes, line counts, and binary detection
- **Multiple sources** → `FileScanner.build_inventory()` → merged `MergeItem` inventory keyed by relative path, categorized as `auto_unique`, `auto_identical`, or `conflict`
- **Inventory** → `MergeEngine.execute_merge()` → files copied to target directory with progress via `queue.Queue` → SSE streamed to browser
- **Browser** → Flask routes → `state` dict (in-memory) + SQLite (persistent) for session config, resolutions, and inventory snapshots
- **Manual merge** → `save_merge()` route → new `FileVersion` with `source_name='Merged'` appended to item's versions, persisted to SQLite
- **Progress events** → `queue.Queue` → SSE endpoints (`scan_events`, `merge_events`) → real-time browser updates

## Invariants Enforced
- Binary files detected by extension (`BINARY_EXTENSIONS` set) or null-byte heuristic; never diffed as text
- Files larger than `MAX_FILE_CONTENT_SIZE` (500 KB) skip content display; files larger than `MAX_LINE_COUNT_SIZE` (10 MB) skip line counting
- Diffs truncated to `MAX_DIFF_LINES` (10,000) unified lines and `MAX_SBS_DIFF_LINES` (8,000) side-by-side lines
- SQLite schema auto-created on first connection via `_init_db_schema`; all tables use `IF NOT EXISTS`
- Conflict items default to the version with the most lines (or largest file size for all-binary conflicts)
- `auto_unique` and `auto_identical` items are always auto-resolved (`resolved=True`)
- Merge execution skips files whose target already has identical SHA-256 content
