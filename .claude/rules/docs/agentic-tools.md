## Purpose
Provides sandboxed file-system operations for the agentic engine. Each tool module exposes a uniform `execute(**kwargs, _base_dir) -> str` interface that returns JSON-encoded results, enabling the engine, MCP server, and workflow builder to dispatch calls through a tool registry. All tools enforce path traversal protection and use only Python standard library dependencies.

## Data Flows
- **Input:** Each `execute()` function receives keyword arguments (tool-specific params + `_base_dir` for sandbox root).
- **Processing:** Path validation resolves and constrains paths within `_base_dir`. Read tools traverse the filesystem with depth limits and skip lists. Write tools create parent directories automatically.
- **Output:** All tools return a JSON string with a response envelope containing results (content, entries, matches, tree text) and metadata (line counts, truncation flags, error messages).

## Invariants Enforced
- **Path sandboxing:** `_validate_path()` in every tool that accepts a path resolves the target and verifies it stays within `_base_dir`. Returns `None` (triggering an error response) on traversal attempts.
- **Depth limiting:** Recursive operations (`list_directory`, `get_file_tree`) cap recursion depth (3 and 4 respectively) to prevent runaway traversal.
- **Result truncation:** `search_code` caps matches at `_MAX_MATCHES = 100` and sets a `truncated` flag.
- **Skip lists:** `get_file_tree` and `search_code` skip common non-source directories (`.git`, `node_modules`, `__pycache__`, etc.) and binary extensions (`.pyc`, `.o`, `.so`, `.dll`, etc.).
