## Purpose
Monolithic Flask application (`file_merger.py`, ~2861 lines) implementing a multi-source file recovery and merge tool. Scans multiple source directories (including Windsurf/VS Code/Cursor editor history), detects conflicts via SHA-256 hashing and content normalization, provides interactive diff/merge editing, and copies resolved files to a target directory. Persistence uses per-session SQLite databases with WAL mode.

## Data Flows
- **Configuration input:** User submits source directories, target directory, and ignore patterns via `/setup` POST → stored in `state` dict and persisted to SQLite via `save_config()`.
- **File scanning:** `FileScanner.build_inventory()` walks source directories in parallel (`ThreadPoolExecutor`, 8 workers) → emits progress events via `queue.Queue` → consumed by `/scan-events` SSE endpoint → results stored in `state['inventory']` and bulk-saved to SQLite via `save_inventory_state()`.
- **Conflict resolution:** User selects versions via `/resolve` POST or `/api/save-merge` POST → single-row UPDATE on `merge_items` table via `save_item_resolution()`.
- **Merge execution:** `MergeEngine.execute_merge()` copies selected file versions to target directory → progress via queue → `/merge-events` SSE → merge log saved as JSON file.
- **Editor history extraction:** `EditorHistoryExtractor.extract_all()` parses `entries.json` from editor history dirs → copies files with original paths to destination → progress via `/extract-events` SSE.

## Invariants Enforced
- **Auto-categorization:** Files with a single version → `auto_unique`; multiple versions with identical SHA-256 → `auto_identical`; differing content → `conflict`.
- **Content normalization:** Files with identical content after line-ending and trailing-whitespace normalization are auto-resolved as `auto_identical` (even if SHA-256 differs).
- **Conflict default selection:** Unresolved conflicts default to the version with the most lines (fallback: largest file size for binary files).
- **Thread-local SQLite:** Each thread gets its own `sqlite3.Connection` via `threading.local()` — connections never cross thread boundaries.
- **WAL mode:** All SQLite databases use `PRAGMA journal_mode=WAL` for concurrent read access during background writes.
- **Legacy migration:** On startup, JSON flat files (`.merger_config.json`, `.merger_state.json`) are automatically migrated to the `sessions/` SQLite structure.
