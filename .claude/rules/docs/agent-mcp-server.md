## Purpose
MCP protocol adapter that exposes the Guanine agent sandbox to external MCP clients (Claude Desktop, Cursor). It wraps session management, file operations, command execution, and agent orchestration behind MCP tools and resources, maintaining global session state to scope all operations to an active workspace.

## Data Flows
- **Client → MCP Server:** External MCP clients invoke tools via stdio or streamable-http transport. Each call hits a `@mcp.tool()` decorated function in `agent_mcp_server.py`.
- **Session scoping:** Most tool calls pass through `_require_session()`, which reads module-level `_active_session` / `_active_repo` globals and injects them into downstream calls.
- **File writes:** `write_file` delegates to `agent_workflow.tracked_write_file` for automatic change tracking within the session.
- **Completion flow:** `signal_done` transitions session status; `reconcile` computes diffs and bridges to the review system.
- **Auto-resume:** On server startup, `_auto_resume()` finds the most recent running session and activates it.

## Invariants Enforced
- `_require_session()` raises `ValueError` if no session is active, preventing unscoped file/command operations.
- `create_session` auto-activates the new session, ensuring subsequent calls are scoped immediately.
- `spawn_agent` creates a child session linked to the current session for parallel execution.

## Impact Notes
- This area is a **pure adapter** — it adds no business logic beyond session scoping and auto-resume. Changes here affect only the MCP client integration surface.
- All downstream dependencies are read-only from this area's perspective: it calls into `agent_schema.py` (imported by 5 files), `agent_tools.py` (imported by 3 files), and `agentic.tools.*` modules. Breaking changes in any of those modules would require updates here.
- The global `_active_session` / `_active_repo` pattern means this server is **not safe for concurrent MCP clients** sharing the same process — each client activation overwrites the previous session binding.
- Removing or renaming any `@mcp.tool()` function changes the external MCP tool API surface, breaking connected clients.
