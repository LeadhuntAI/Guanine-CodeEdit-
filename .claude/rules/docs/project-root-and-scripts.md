## Purpose
This area serves as the project's entry point and operational metadata layer. `CLAUDE.md` is the central discovery hub for AI agents, implementing a decentralized documentation protocol that directs agents to module-specific `RULES_INDEX.md` files. `restore_deleted_files.ps1` provides a standalone CLI alternative to the web UI for file recovery from editor local history. The `sessions/` directory is the runtime persistence layer for per-session SQLite databases.

## Data Flows

- **AI agent discovery**: Agents read `CLAUDE.md` first, then follow the rule discovery protocol to load module-specific rules from `.claude/rules/` and `.claude/RULES_INDEX.md`.
- **CLI file recovery**: `restore_deleted_files.ps1` scans editor history directories (Windsurf, VS Code, Cursor), parses `entries.json` files, resolves backup files, and copies them to a destination folder — all without the Flask app.
- **Session persistence**: `core-application` (`file_merger.py`) writes per-session SQLite databases into `sessions/` at runtime.

## Invariants Enforced

- **No silent file destruction** (documented in CLAUDE.md, enforced in `restore_deleted_files.ps1` via confirmation prompt and `-Force` flag requirement).
- **Session persistence**: Session data must always be persisted to SQLite in `sessions/`, never held only in memory.

## Configuration

- **No `.env` file** — app uses hardcoded defaults (port 5000, Flask secret key in source).
- **Session storage** — `sessions/` directory, each session gets its own SQLite DB.
- **Editor history paths** — hardcoded in `restore_deleted_files.ps1` for Windows (9 paths across Windsurf, VS Code, Cursor).
- **CLI parameters** — `restore_deleted_files.ps1` accepts `-ProjectFolder` (required), `-DestinationFolder`, `-ExcludeFolder`, `-Hours` (default 120000), `-DryRun`, `-Force`, `-show_all`.

## Impact Notes

- **`sessions/` directory**: If removed or relocated, `core-application` will fail to persist session data. The path is hardcoded in `file_merger.py`.
- **`CLAUDE.md`**: This is the single entry point for all AI agent interactions. Changes to the rule discovery protocol or directory layout documentation will affect how agents navigate the project.
- **`restore_deleted_files.ps1`**: Independent of the Flask app — changes here don't affect the web UI, but the script reimplements logic from `EditorHistoryExtractor` in `file_merger.py`, so bug fixes in one may need to be mirrored in the other.
- **`.gitignore`**: Controls what runtime artifacts are tracked. Adding new runtime files (e.g., new DB locations) requires updating this file.
