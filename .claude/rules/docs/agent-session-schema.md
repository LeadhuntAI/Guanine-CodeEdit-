## Purpose
Core data layer providing SQLite-backed persistence for the agent system. Manages repos, sessions, file checkouts, conversation history, and review decisions through a repository-pattern API with thread-local connection pooling and WAL journaling. This is the most-imported module in the codebase (5 importers), serving as the single source of truth for all agent state.

## Data Flows
- **Repo registration** — `register_repo()` receives repo config (path, name, ignore patterns, command whitelist), generates a deterministic SHA-256 ID, and persists to `repos` table.
- **Session lifecycle** — `create_session()` creates a workspace directory and session record; `update_session_status()` advances through lifecycle states (`pending` → `running` → `completed` → `review` → `merged`/`rejected`), automatically setting timestamp fields (`started_at`, `completed_at`, `reviewed_at`, `merged_at`).
- **File tracking** — `record_file_checkout()` logs files pulled into workspace; `update_file_stats()` and `record_new_file()` track modifications with diff statistics; `get_session_files()` and `get_modified_files()` query file state.
- **Conversation history** — `save_conversation_message()` appends LLM messages with optional tool call metadata; `get_conversation()` retrieves full history.
- **Review decisions** — `record_review_decision()` upserts per-file review outcomes; `get_review_summary()` aggregates progress counts.
- **All data exits** as plain `dict` objects (no ORM), with JSON-serialized complex fields deserialized by `_deserialize_repo()` and similar helpers.

## Invariants Enforced
- **Session status** must be one of `VALID_SESSION_STATUSES`: `pending`, `running`, `completed`, `review`, `merged`, `rejected`.
- **Review decisions** must be one of `VALID_REVIEW_DECISIONS`: `accepted`, `rejected`, `edited`, `reverted`.
- **Foreign key constraints** enforced at the SQLite level — child records (files, conversation, reviews) reference valid session/repo IDs.
- **Cascade deletion** — `_delete_session_data()` removes all child records before deleting a session.
- **UPSERT semantics** — `record_file_checkout()` and `record_review_decision()` use `INSERT ... ON CONFLICT` for idempotent operations.
- **Thread safety** — SQLite connections are cached per-thread via `threading.local()`; WAL mode enables concurrent reads.
- **Deterministic repo IDs** — SHA-256 of normalized path ensures the same repo always maps to the same ID.
