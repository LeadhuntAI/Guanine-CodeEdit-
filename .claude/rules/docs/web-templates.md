## Purpose
Complete Jinja2 template suite for the Flask-based File Recovery Merger application. Provides a Bootstrap 5 dark-themed UI covering the full workflow: setup wizard, interactive file browser with conflict resolution, inventory/conflict list views, hunk-based merge editor, coverage dashboard, editor history extraction, and real-time progress pages via Server-Sent Events.

## Data Flows
- **Flask routes → templates**: Route handlers pass template variables (stats, items, versions, diffs, coverage data, log entries) via `render_template()`. All templates extend `base.html`.
- **Templates → Flask API (AJAX)**: `browse.html` and `_file_detail.html` call `/api/browse-tree`, `/api/file-detail/*`, `/api/file-content/*`, `/api/save-merge`, `/api/coverage-files/*` for dynamic content loading.
- **Templates → Flask SSE**: `scan_progress.html`, `merge_progress.html`, `extract_progress.html` consume Server-Sent Events from `/scan-events`, `/merge-events`, `/extract-events` respectively.
- **Templates → Flask POST**: Forms and JS functions POST to `/setup`, `/execute`, `/resolve`, `/resolve-toggle`, `/resolve-accept-defaults`, `/resolve-all-latest`, `/extract-history` for state mutations.
- **Flash messages**: `base.html` renders Flask flash messages with auto-dismiss via `get_flashed_messages()`.

## Invariants Enforced
- All page templates extend `base.html` (except partial `_file_detail.html`).
- Jinja2 block contract: child templates fill `title`, `content`, and optionally `extra_css`/`extra_js`.
- Bootstrap 5 dark theme is applied globally via `data-bs-theme="dark"` on `<html>`.
- Unsaved changes warning in `merge_editor.html` prevents accidental navigation.

## Impact Notes
- **core-application** is the sole consumer. Any change to template variable names (e.g., renaming `items` → `files`) requires updating the corresponding `render_template()` calls in `file_merger.py`.
- **API contract**: `browse.html`, `_file_detail.html`, `merge_editor.html`, and `coverage.html` depend on specific JSON response shapes from `/api/*` endpoints. Changing API response schemas will break these templates silently (no build-time check).
- **SSE event format**: The three progress templates (`scan_progress.html`, `merge_progress.html`, `extract_progress.html`) parse SSE messages by event type. Changing event names or payload structure in the server will break real-time progress display.
- **Bootstrap 5 dependency**: All templates assume Bootstrap 5 classes and dark theme. Upgrading Bootstrap major versions would require class updates across all templates.
