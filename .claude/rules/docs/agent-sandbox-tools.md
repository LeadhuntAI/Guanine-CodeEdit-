## Purpose
Provides the core sandboxed execution environment for AI coding agents. Agents checkout files from a repository into an isolated workspace, make modifications, run commands, and signal completion for human review. The area enforces security through path validation, command whitelisting, and resource limits, while tracking changes via SHA-256 hashes for diff computation and review integration.

## Data Flows
- **File checkout:** `checkout_file` / `checkout_files` copy files from repo → workspace, recording SHA-256 hashes in `session_files` via `agent_schema.record_file_checkout`.
- **File writes:** `tracked_write_file` wraps `agentic.tools.write_file`, updating `session_files` with diff stats and new hashes.
- **Command execution:** `run_command` executes shell commands in the workspace directory, enforcing command whitelists and truncating output.
- **Session finalization:** `signal_done` marks a session completed; `reconcile_session` computes diff stats, detects new/deleted files, and bridges to the review system.
- **Multi-agent spawning:** `spawn_agent` creates child sessions with isolated workspaces and parent-child linking.
- **Tool registry:** `build_tool_registry` binds all tools to a specific session/workspace via `functools.partial`, producing a dict consumed by the agentic engine.

## Invariants Enforced
- **Path containment:** `_validate_path` ensures all file paths resolve within `base_dir`, preventing directory traversal.
- **Command whitelist:** `run_command` checks commands against `repo.allowed_commands` unless `allow_free_commands` is enabled.
- **Resource limits:** Output truncated at `_MAX_STDOUT` (10,000 chars) and `_MAX_STDERR` (5,000 chars); file listing capped at `_MAX_FILE_RESULTS` (500).
- **JSON return contract:** All tool functions return JSON strings for consistent interop with the agentic engine.
- **Hash-based integrity:** SHA-256 hashes recorded at checkout are compared on reconciliation to detect modifications.
