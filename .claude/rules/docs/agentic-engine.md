## Purpose
Lightweight autonomous AI workflow engine implementing a layered architecture for LLM-powered automation. It orchestrates multi-layer workflows (prompt, agentic loop, template, tool), provides dual-mode agentic loops (ReAct text-parsing and native OpenAI function-calling), a stdlib-only OpenRouter HTTP client with retry logic, safe tool execution with signature filtering, and a knowledge resolution system for assembling rules/skills into prompts.

## Data Flows
- **Workflow execution:** `run_workflow()` receives a workflow JSON dict → iterates layers → `process_layer()` dispatches by `layer_type` → each layer returns `{response, ...}` → session state accumulates → final `session.to_dict()` returned.
- **Agentic loop (ReAct):** System prompt assembled from layer def + knowledge → LLM called with stop token `<END_OF_ACTION>` → response parsed for `Action:`/`Answer:` patterns → tool calls dispatched via `execute_tool_call()` → observations appended → loop until answer or max iterations → `summarize_loop()` produces structured summary.
- **Agentic loop (native):** Same flow but uses OpenAI-format `tools` parameter → LLM returns `tool_calls` in response → executed and fed back as `role: tool` messages.
- **Knowledge resolution:** `resolve_knowledge()` reads rule files (full text), discovers skills from `skills/*/SKILL.md` and `skill_definitions/*.json`, builds XML-wrapped indexes → injected into system prompts via `_build_system_message()`.
- **Tool execution:** `execute_tool_call()` receives tool registry + name + args → `parse_tool_args()` normalizes string args → `_filter_args()` introspects function signature → filtered args passed to callable → result JSON-serialized.

## Invariants Enforced
- `execute_tool_call` never raises — all errors are caught and returned as `{"error": "..."}` JSON strings.
- `_filter_args` strips arguments not present in the target function's signature (unless `**kwargs` is accepted), preventing injection of unexpected parameters.
- `OpenRouterClient._post` retries only on 429/5xx/URLError; 4xx client errors (except 429) raise immediately.
- `extract_json` and `parse_tool_args` return `None`/`{}` on failure rather than raising.
- `_safe_summarize` wraps `summarize_loop` to never propagate exceptions.
- Jinja2 rendering uses `BaseLoader` (no filesystem access) and returns the unprocessed template string on error.
