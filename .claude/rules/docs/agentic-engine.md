## Purpose
Standalone workflow execution engine for LLM-powered agents. Workflows consist of multiple processing layers (prompts, agentic loops, templates, tool calls) that accumulate state in a session object. Supports dual-mode agentic execution: ReAct-style text parsing and native OpenAI-format function calling. Designed with stdlib-only dependencies where possible, with robust error handling and defensive JSON parsing for unreliable LLM outputs.

## Data Flows
- **Workflow input** enters via `run_workflow()` in `runner.py`, which iterates through layer definitions sequentially.
- **Session state** (`LightweightWorkflowSession`) accumulates context, chat history, and layer outputs across layers.
- **LLM calls** go through `OpenRouterClient` (`openrouter.py`) → external OpenRouter API, with retry/backoff and response normalization.
- **Tool dispatch** flows through `execute_tool_call()` (`tool_executor.py`) → functions in the `tool_registry` dict (populated at runtime by `agentic-tools`).
- **Knowledge assembly** flows through `resolve_knowledge()` (`knowledge.py`) → filesystem scan for rules/skills → XML-wrapped indices injected into system prompts.
- **OpenCode integration** flows through `OpenCodeClient` (`opencode_client.py`) → external OpenCode server process via HTTP/SSE.

## Invariants Enforced
- Agentic loops are bounded by `_DEFAULT_MAX_ITERATIONS` (20) to prevent infinite loops.
- `OpenRouterClient._post()` retries up to `_MAX_RETRIES` (3) with exponential backoff `(2, 4, 8)` seconds.
- `execute_tool_call()` filters arguments via `_filter_args()` to match the target function's signature, preventing `TypeError` from unexpected kwargs.
- `extract_json()` uses brace-counting fallback (`_extract_braced_json`) for malformed LLM JSON output.
- `LightweightWorkflowSession` maintains ordered `chat_history` and `outputs` lists for deterministic layer sequencing.

## Configuration
- **`OPENROUTER_API_KEY`** — Required by `OpenRouterClient` for LLM API authentication.
- **`OpenRouterClient.base_url`** — Defaults to `https://openrouter.ai/api/v1`; configurable via constructor.
- **`OpenCodeClient.base_url`** — Defaults to `http://127.0.0.1:4096`; configurable via constructor.
- **`_DEFAULT_MAX_ITERATIONS`** — Agentic loop iteration cap (default 20), defined in `loop.py`.
- **`_MAX_RETRIES` / `_BACKOFF_SECONDS`** — Retry config for OpenRouter HTTP calls (3 retries, 2/4/8s backoff), defined in `openrouter.py`.
- **`_HEALTH_TIMEOUT` / `_HEALTH_POLL_INTERVAL`** — OpenCode server health check config (30s timeout, 0.5s poll), defined in `opencode_client.py`.

## Impact Notes
- **`OpenCodeClient`** is imported by `agent-sandbox-tools` and `agent-review-and-git`. Changes to its public API (`health_check`, `ensure_server`, `create_session`, `send_message`, `stream_events`, `get_diff`, `list_agents`) would break those areas.
- **`LightweightWorkflowSession`** is a shared type used by `agent-sandbox-tools`. Changes to its attributes or methods would affect downstream workflow orchestration.
- **`execute_tool_call`** in `tool_executor.py` is the dispatch point for all tool execution. Changes to argument filtering or error handling would affect every tool call across all workflows.
- **`OpenRouterClient._normalise`** defines the internal response format consumed by both `process_agentic_loop` and `process_agentic_loop_native`. Changes to the normalized dict shape would break agentic loop execution.
